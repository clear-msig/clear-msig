## Evidence

- Root `Cargo.toml` lists the program and client as workspace members.
- `.github/workflows/ci.yml` runs `quasar build`, uploads
  `target/deploy/clear_wallet.so`, and executes SVM tests against it.
- `clear-msig-execution` and `apps/e2e` import the generated modules from
  `clear-wallet-client`; neither keeps a private instruction snapshot.
- Frontend account parsers, PDA derivation, error mapping, and policy views
  mirror the active program layouts.
- Deployment and product documents identify the same devnet program.

## Removal decisions

The two stale generated snapshots under `crates/clear-msig-execution` and
`apps/e2e` were removed after fresh generation showed both had drifted from the
active program ABI. Imports now use the program-owned generated client.

Program-local Cargo lockfiles and the package-local Quasar manifest were also
removed. `cargo metadata` resolves both program packages to the repository root
workspace and target directory; CI and deploy scripts run the root
`Quasar.toml`. No second Solana program directory was found, so no active
program source was deleted merely because it was old.

`scripts/check-generated-client.sh` rejects generated drift and duplicate
client directories after every CI `quasar build`.
